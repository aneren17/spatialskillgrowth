# 异常检测服务接口

本服务提供基于多模态模型的异常事件检测能力。支持对图片和短视频进行指定类别的异常判断。服务在两个独立的端口上提供了完全相同的检测接口，可根据需要选择调用。

## 1. 接口定义

```text
POST http://172.16.0.91:<18061|18062>/detect
Content-Type: multipart/form-data

```

### 输入字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `file` | 文件 | 是 | 一张图片或一个短视频。文件名必须包含受支持的扩展名。 |
| `event_type` | 字符串 | 是 | 已确定的异常类别。推荐使用精确英文 ID，例如 `banner`、`fire`、`fall`。 |

**支持的图片扩展名：**

```text
.bmp .jpeg .jpg .png .webp

```

**支持的视频扩展名：**

```text
.avi .m4v .mkv .mov .mp4 .mpeg .mpg .webm

```

> **说明：** 默认单文件上限为 256 MiB。服务不负责重新判断异常类别，只严格执行 `event_type` 指定类别的检测。

### 调用示例

**18061 端口调用示例：**

```bash
curl -X POST http://172.16.0.91:18061/detect \
  -F "file=@test/banner.mp4" \
  -F "event_type=banner"

```

**18062 端口调用示例：**

```bash
curl -X POST http://172.16.0.91:18062/detect \
  -F "file=@test/banner.mp4" \
  -F "event_type=banner"

```

---

## 2. 响应格式

### 成功输出 (HTTP 200)

```json
{
  "is_anomaly": 1,
  "threshold": 0.66
}

```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `is_anomaly` | 整数 | `1` 表示检测到异常，`0` 表示未检测到异常。 |
| `threshold` | 浮点数 | embedding 返回的判定阈值；其他图片工作流覆盖为异常时返回 `1.0`。 |

> **执行逻辑：** 视频推理会并行执行原视频 embedding 工作流和检索到的图片工作流，并用确定性 OR 汇总：任一有效工作流判断异常，最终 `is_anomaly` 即为 `1`。

### 错误输出

错误响应使用统一的 JSON 格式：

```json
{
  "detail": "错误原因"
}

```

| HTTP 状态码 | 常见原因 |
| --- | --- |
| `400` | `event_type` 不受支持、文件扩展名不受支持、文件为空或超过大小限制。 |
| `422` | 缺少 `file` 或 `event_type` 表单字段。 |
| `500` | 服务内部异常检测执行失败。 |
| `502` | 下游模型或工具没有返回可接受的异常判断或判定阈值。 |
